About TellMyLocation

Engineering Transparency in Geolocation

TellMyLocation.com is not just a utility; it is a proof-of-concept for Privacy-First Engineering. In an ecosystem dominated by mobile apps that aggressively harvest, store, and monetize user movement history, we built the antithesis: a stateless, client-side tool that delivers military-grade precision without the surveillance.

The Architecture of Privacy

Most "Where Am I" services operate on a server-centric model: your device sends its coordinates to a remote server, the server processes the map, and sends an image back. This creates a permanent log of your location on someone else's computer.

We rejected this model. Instead, we use a Client-Side Architecture:

  • Local Execution: The JavaScript code that calculates your position runs entirely inside your browser's memory (RAM).
  • Direct Hardware Access: We interface directly with the navigator.geolocation API standard, allowing your browser to talk to your device's GPS chip without a middleman.
  • Ephemeral Sessions: There is no database. There are no user accounts. When you close the tab, the data instance is destroyed instantly.

Technical Specification: W3C Geolocation Standard

We strictly adhere to the W3C High-Accuracy protocols. When you grant permission, we request a PositionOptions object with enableHighAccuracy: true. This forces the device to prioritize satellite trilateration over generic Wi-Fi triangulation, ensuring 3-meter accuracy rather than the standard 100-meter block-level approximation.

How It Works: The "Tri-Hybrid" Lock

To provide the fastest possible fix, our tool utilizes a hybrid positioning system that layers three distinct technologies:

  1. A-GPS (Assisted GPS): Uses cellular tower data to download satellite almanacs faster, reducing "Time to First Fix" (TTFF) from minutes to milliseconds.
  2. Wi-Fi Fingerprinting: In dense urban environments (Urban Canyons) where skyscrapers block the sky, we utilize local Wi-Fi SSIDs to triangulate position relative to known routers.
  3. Barometric Altimetry: On supported devices, we access the internal pressure sensor to determine altitude, helping distinguish whether you are on the ground floor or the 50th floor.
